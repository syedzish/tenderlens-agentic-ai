# Agents CLI Specification - TenderLense Agentic AI

## Purpose

TenderLense Agentic AI is an ADK 2.0 multi-agent system that helps a bidding company decide whether to bid for a tender. It analyzes tender evidence against a bidder profile, uses OKF + RAG for grounded retrieval, coordinates parallel specialist agents, audits the draft through a mandatory A2A Remote Evidence Audit Agent, and returns a cited bid/no-bid decision through typed and voice modes.

## Required Stack

- Python ADK project scaffolded or enhanced through Agents CLI.
- Main backend deployment target: Agent Runtime.
- Real-time voice: ADK Gemini Live API Toolkit.
- Voice model configured by `VOICE_AGENT_MODEL`.
- Frontend: Vercel-hosted web app.
- MCP: app-local `tenderlense-mcp` server.
- Data format: OKF markdown bundles with YAML frontmatter and citations.
- Evals: Agents CLI evals plus deterministic unit/integration/conformance tests.
- Observability: Cloud Trace where supported by deployment path.

## Main Agent

Name: `tenderlense-router`

Responsibilities:

- Accept typed input or voice transcript turns.
- Validate selected tender and bidder profile context.
- Select sample OKF bundle or trigger transient uploaded-tender processing.
- Use MCP tools for tender, profile, evidence, scenario, upload, and voice context.
- Coordinate parallel specialist analysis.
- Call Synthesis Agent.
- Send the draft package to the A2A Evidence Audit Agent.
- Run the Bounded Evidence Quality Loop before final response.
- Return structured dashboard payload and concise voice payload when needed.

## Mandatory A2A Remote Evidence Audit Agent

The audit agent must be exposed through ADK A2A support and consumed by the Router/Synthesis workflow before final output.

Audit input:

- Draft recommendation
- Specialist outputs
- Evidence package
- Citations
- Language mode
- Voice response payload when active
- Privacy/upload/voice flags

Audit output:

- `status`: `pass`, `fail`, or `unavailable`
- Missing citations
- Unsupported claims
- Weak evidence
- Schema issues
- Arabic-mode issues
- Voice response grounding issues
- Privacy/logging concerns
- Revision instructions

If the A2A audit is unreachable, the product must surface "audit unavailable" instead of pretending the audit passed.

## Bounded Evidence Quality Loop

Loop flow:

```text
synthesis draft
  -> A2A audit
  -> pass: final response
  -> fail: router sends issue to targeted existing specialist or retrieval
  -> specialist/retrieval fixes section
  -> synthesis rebuilds
  -> A2A audit again
  -> stop after pass or max 2 revision rounds
```

The Targeted Specialist Agent is not a new agent. It means the existing specialist responsible for the failed section.

Retrieval joins only when missing or weak evidence caused the audit failure.

## MCP Server

Name: `tenderlense-mcp`

Tools:

- `list_tenders`
- `get_tender_okf_index`
- `get_okf_concept`
- `search_evidence`
- `get_company_profile`
- `score_profile_against_requirement`
- `get_scoring_rubric`
- `simulate_strategy_overlay`
- `generate_clarification_question_candidates`
- `validate_okf_bundle`
- `get_upload_analysis_status`
- `get_voice_session_context`

MCP rules:

- Use strict tool filtering.
- Validate all inputs with schemas.
- Return structured outputs.
- Include timeouts.
- Avoid broad filesystem access.
- Do not expose upload temp paths.
- Keep curated samples read-only.
- Keep uploaded and voice context transient.

## Voice Mode

Voice mode is a core v1 feature and must start only by explicit user action.

Implementation direction:

- Browser captures microphone audio only after Voice Mode click.
- WebSocket connects to a thin Voice Gateway.
- Voice Gateway sends audio/text into ADK Live `LiveRequestQueue`.
- Agent streaming uses `runner.run_live()` and `RunConfig`.
- Stream model audio, transcript, dashboard, and audit events back to the browser.
- Do not store or log raw audio.
- Enforce `VOICE_SESSION_MAX_SECONDS`, default 300.
- Typing mode remains the guaranteed fallback.

Voice event categories:

- `session.start`
- `session.ready`
- `audio.chunk`
- `audio.end`
- `transcript.partial`
- `transcript.final`
- `agent.text.partial`
- `agent.audio.chunk`
- `agent.turn.complete`
- `agent.interrupted`
- `dashboard.patch`
- `audit.status`
- `error`
- `session.end`

## Upload Constraints

- Max file size: 5 MB.
- Supported types: PDF, TXT, MD, DOCX when parser support is stable.
- Uploads are processed server-side only.
- Uploaded file bytes, raw extracted text, and generated uploaded OKF bundles are transient.
- No upload temp paths are exposed to browser or MCP tools.
- Curated samples remain the reliable public demo path.

## Security And Privacy

Required controls:

- No secrets in code.
- `.env.example` only.
- Server-side credentials only.
- Sanitized rendering for model output, tender text, and transcripts.
- Prompt injection guardrails around tender evidence.
- MCP tool filtering and schema validation.
- Bounded loops.
- Citation-required outputs.
- Privacy-safe logging.
- Raw uploads and raw audio not retained.

Public claims:

- "We do not save your uploaded files."
- "We only listen when you start voice mode, and we do not save your voice."

## Bilingual Requirements

- English default.
- Arabic full UI/content mode with RTL layout.
- Same analysis available in both languages.
- Arabic voice mode when supported by selected Live API model.
- If Arabic spoken output is unavailable, Arabic typed mode remains fully supported and UI states the limitation.

## Eval Success Criteria

- Correct bid/no-bid recommendation for sample scenarios.
- Citations present for material claims.
- Router uses MCP before final claims.
- Router invokes A2A audit before final response.
- Missing citations trigger the quality loop.
- Max 2 revision rounds respected.
- Voice transcript routes to same workflow.
- Prompt injection in tender text is ignored as an instruction.
- Arabic mode returns Arabic output and RTL UI.
- Upload >5 MB rejected before parsing.

## Deployment Expectations

- Backend deploys to Agent Runtime only after explicit user approval.
- Frontend deploys to Vercel.
- Voice gateway uses Vercel WebSockets/Fluid Compute if reliable; Cloud Run fallback requires explicit approval.
- Vercel routes call backend/voice services server-side.
- No `NEXT_PUBLIC_*` secrets for Agent Runtime or Live API credentials.
- Public demo does not require login or paywall.
